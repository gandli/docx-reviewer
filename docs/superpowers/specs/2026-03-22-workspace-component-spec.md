# 主工作区组件级规范

**项目：** 浏览器本地离线文书工作台  
**日期：** 2026-03-22  
**状态：** 已确认，可指导实现

## 1. 目标

把主工作区进一步细化到组件层，确保后续实现时：

- 左栏、中栏、右栏职责不混
- “文书生成”和“文书审阅”两条主线在组件层可被清楚承载
- 高端产品感不会在实现阶段走样
- “工作状态”可以作为首版最先打磨的真实页面

## 2. 页面骨架

主页面固定为三栏：

- 左栏 `22%`
- 中栏 `54%`
- 右栏 `24%`

允许用户拖拽微调，但默认比例不变。

补充约束：

- 页面外层贴边满屏
- 不再包一层大圆角外框
- 视觉层级主要由分栏、留白和中栏纸张感建立
- 页面整体不滚动
- 左栏、中栏、右栏都使用独立滚动区域
- 滚动条应尽量隐形

## 3. 左栏组件

### 3.1 `WorkspaceHeader`

作用：

- 显示当前工作区名称
- 显示最近更新时间
- 提供工作区级身份感

内容：

- 工作区名称
- 最近更新时间
- 可选的轻量状态说明

要求：

- 字重稳，不夸张
- 信息少，但要有存在感

### 3.2 `AssetGroupList`

作用：

- 承载工作区材料树

默认分组：

- 主文档
- 模板
- 参考资料
- 表格资料

要求：

- 分组层级比后台导航更轻
- 不使用过多图标
- 当前活动文档用柔和高亮，不用强对比色
- 使用分隔线和字重建立层级，尽量少用卡片容器
- 每个分组都支持折叠与展开
- 默认展开当前主文档
- 分组按钮应只承担展开收起，不做重卡片样式
- 展开后的子项更像安静的资料列表，而不是第二层导航按钮
- 分组标题显示数量
- 子项显示名称、类型和最近更新时间
- 当前选中项只做低对比强调
- 展开指示器使用更细的箭头，不使用 `+ / −`

### 3.3 `RecentEvidenceList`

作用：

- 展示最近使用的引用证据

内容：

- 来源标题
- 位置
- 简短描述

要求：

- 只显示少量高价值项
- 更像“最近参考过什么”，不是完整日志

### 3.4 `WorkspaceActions`

作用：

- 集中放导入、导出、设置

要求：

- 放在左栏底部
- 不能挤占主体区域
- 设置应打开遮罩层，而不是在左栏内展开二级面板
- 设置层首版至少承载：
  - 提示词偏好
  - 主题色
  - 本地模型

## 4. 中栏组件

### 4.1 `DocumentHeader`

作用：

- 显示当前主文档名称
- 显示阅读态 / 可编辑状态

内容：

- 文档标题
- 轻量状态文本
- 可选的保存状态

要求：

- 视觉要克制
- 不能像传统工具栏一样过重
- 状态信息应像文档页眉注记，而不是工具标签

### 4.2 `DocumentCanvas`

作用：

- 承载整份主文档的纸张化显示

要求：

- 中栏必须是全页面最有质感的区域
- 页面宽度、边距、阴影和留白要体现纸张感
- 不能像普通富文本编辑器那样“整页都是控件”
- 建议数量提示应以轻边注形式出现

### 4.3 `InlineSelectionBlock`

作用：

- 展示当前被选中的条款或段落

表现：

- 柔和边框
- 温和底色
- 与普通正文形成明确但不刺眼的区分

### 4.4 `RiskMarker`

作用：

- 在文档中标出风险、建议或待处理位置

表现：

- 细边线、轻标题、边注提示
- 不做大红大绿

要求：

- 风险可见，但不破坏阅读
- 更像文档修订批注，而不是功能卡片

### 4.5 `InlineEditState`

作用：

- 用户点击后原位进入编辑态

要求：

- 编辑态必须仍然看起来属于文档本身
- 不弹出另一套大编辑器覆盖页面
- 保存后回到阅读态，保持页面稳定

## 5. 右栏组件

### 5.1 `AssistantContextHeader`

作用：

- 告诉用户当前助手正在围绕哪个上下文工作

内容：

- 当前上下文标题
- 当前任务轻量状态
- 是否已恢复工作区摘要

要求：

- 保持极简，不形成大信息卡
- 只保留当前阅读或编辑最需要的一行上下文
- 当前实现应优先保留“一行上下文 + 当前条款”

### 5.2 `AssistantMessageList`

作用：

- 显示连续对话消息

要求：

- 语气专业克制
- 以消息线程呈现
- 需要区分用户消息与助手消息
- 风格接近 ChatGPT，但更克制、更适合专业文档场景

### 5.3 `ChatComposer`

作用：

- 提供底部固定输入区

内容：

- 输入占位提示
- 发送按钮
- 模型设置入口

要求：

- 固定在右栏底部
- 是右栏的主要交互入口
- 风格接近 ChatGPT 输入区，但颜色和材质保持暖中性
- 模型设置应以图标入口融入输入区，不单独占用一整行

### 5.4 `EvidenceMessage`

作用：

- 展示引用来源和证据说明

内容：

- 来源文件
- 页码或表格位置
- 简短说明

### 5.6 `ActionPanel`

作用：

- 对当前选中内容执行动作

默认动作：

- 接受建议
- 重新生成
- 跳到原文位置

要求：

- 不再常驻展示
- 通过“更多操作”以浮动层方式打开
- 主按钮只有一个
- 其他动作弱化
- 不能把右栏做成满屏按钮墙
- 浮层应尽量轻，不像独立弹窗页面

## 6. 页面状态规范

### 6.1 空工作区

重点组件：

- `WorkspaceHeader`
- `EmptyImportPrompt`
- `AssistantIntro`

要求：

- 更像精致引导，而不是空白后台

### 6.2 工作状态

重点组件：

- `DocumentCanvas`
- `InlineSelectionBlock`
- `AssistantMessageList`
- `ChatComposer`
- `ActionPanel`

这是首版优先状态。

### 6.3 审阅结果状态

重点组件：

- `RiskMarker`
- `ReviewSummaryCard`
- `EvidenceMessage`

要求：

- 仍然保留同一主工作区，不新开页面

## 7. 首版实现优先级

1. `WorkspaceHeader`
2. `WorkspaceResumeCard`
3. `AssetGroupList`
4. `DocumentHeader`
5. `DocumentCanvas`
6. `InlineSelectionBlock`
7. `AssistantContextHeader`
8. `SuggestedActions`
9. `AssistantMessageList`
10. `ChatComposer`
11. `ActionPanel`

## 8. 验收标准

- 用户重新进入时，能直接在左栏看到“继续上次工作”
- 用户能在中栏一眼识别当前正在处理的文档位置
- 用户能在右栏用连续对话方式继续任务，而不是重新找入口
- 三栏之间不存在职责重叠
- 中栏始终保持主位，左右栏不会抢视觉中心
- 左右两侧以轻分隔为主，而不是卡片堆叠
